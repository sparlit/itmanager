## SECTION 1: COMPLETE WEBSITE PLAN & ARCHITECTURAL BLUEPRINT

### 1.1 Architectural Philosophy: The Federated Monorepo
The system is built as a Federated Monorepo (Next.js 15, PostgreSQL 16, Prisma, Redis) comprising 18+ standalone-style web applications. This "Plug-and-Play" architecture ensures that adding or removing a portal has zero side effects on the rest of the system.

### 1.2 Site Structure & Portals
- **Public E-Commerce Portal:** B2C/B2B interface with dual LTR/RTL support.
- **B2B Corporate Dashboard:** White-labeled portal for high-volume partners (Hotels/Hospitals).
- **Vendor Portal:** Supply chain management for detergent and consumable suppliers.
- **15 Department Portals:** Highly specialized internal applications (Administration, Sales, Marketing, Customer Service, Production, Operations, Transport, HR, IT, Maintenance, Finance, Business Development, Communications, Security, Housekeeping).

### 1.3 Site Map & Conversion Goals
| Portal / Page | Primary Content | Conversion/Action Goal |
| :--- | :--- | :--- |
| Landing Path | Selector: Residential vs Internal Login. | Correct user routing. |
| Public Home | Hero, Trust, Services, Social Proof, FAQ. | Click "Schedule Pickup". |
| B2B Dashboard | Contract stats, Bulk Invoicing, Batch Status. | Request Urgent Batch. |
| Production App | RFID Scan, QC Check, AI Stain Detector. | Zero Defects / High Speed. |
| Transport App | Route Map, Proof of Attempt (Photo/GPS). | On-time Delivery. |
| IT Portal | System Health, PITR Status, API Monitor. | 100% System Uptime. |
