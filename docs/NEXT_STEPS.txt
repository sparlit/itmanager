# PROJECT ASSEMBLY & NEXT STEPS GUIDE
# Laundry ERP & E-Commerce (Qatar)

Following the delivery of the architecture and core modules, here is your roadmap to getting the system live and the recommended next steps for development.

================================================================================
1. IMMEDIATE NEXT STEPS (ASSEMBLY)
================================================================================
Since this is a new project, you should assemble the files provided in the following order:

1.  **Initialize Monorepo Structure:**
    Create the folders: `apps/public-portal`, `apps/internal-portals`, `packages/db`, `packages/logic`, `packages/ui`.

2.  **Database Setup:**
    - Place `packages/db/prisma/schema.prisma` in its folder.
    - Run `npx prisma db push` to create the 15-department schema in your local PostgreSQL.

3.  **Authentication & Logic:**
    - Place `packages/logic/auth/auth-service.ts` into its folder.
    - This provides the "Zero-Stub" JWT and Bcrypt logic used by all portals.

4.  **Internal Portal Launch:**
    - Deploy the `Navigator` (`apps/internal-portals/app/dashboard/page.tsx`) and the `SciFiLogin` component.
    - This allows you to verify the Edex-UI aesthetic immediately.

5.  **Public Portal Pickup Test:**
    - Deploy the `OrderPage` and `QatarLocationPicker`.
    - Verify that you can drop a pin on the map and capture coordinates for Doha/West Bay/Lusail.
