================================================================================
1. ARCHITECTURAL VISION
================================================================================
A resilient, modular, and AI-driven ecosystem comprising 17+ standalone-style web applications. Designed for the high-end Qatar market, the system prioritizes **Disaster Recovery (PITR)**, **WhatsApp-First Communication**, **Strict MFA Security**, and **Dual-Calendar Support**.

================================================================================
2. TECHNOLOGY STACK (100% FOSS)
================================================================================
- **Framework:** Next.js 15 (App Router) + PWA (Offline-First).
- **Database:** PostgreSQL 16 + WAL-G (for PITR) + MinIO (S3-Storage).
- **Auth:** Jose (JWT) + TOTP MFA (Multi-Factor) + Bcrypt.js.
- **Comms:** WhatsApp Business API (via generic abstraction) + SMS Fallback.
- **AI Agents:** LangGraph / CrewAI for Logistics, Sales, and QC.
- **Calendars:** Umm al-Qura (Hijri) + Gregorian Dual Support.
- **DevOps:** GitHub Actions / GitLab CI + Docker + PM2.
- **Privacy:** AES-256 Field-Level Encryption (Prisma Middleware).

5. MISSION-CRITICAL WORKFLOWS
================================================================================

### A. DISASTER RECOVERY (PITR)
- **Engine:** WAL-G continuously streams PostgreSQL Write-Ahead Logs to a local MinIO bucket.
- **Capability:** In the event of a "Midnight Crash," the Admin can restore the DB to a state within seconds of the failure.

### B. WHATSAPP-FIRST COMMUNICATION
- **Event:** `Order.status === 'READY_FOR_DELIVERY'`.
- **Action:** Comms Layer sends a WhatsApp message with a live tracking link.
- **Fallback:** If WhatsApp API returns an error, the system automatically sends a standard SMS.

### C. CLAIMS & LIABILITY MANAGEMENT
- **Workflow:** Customer initiates a claim via Public Portal -> "Claims Module" alerts Admin -> Admin reviews Production/QC logs for that Item -> Compensation (Credit/Refund) is authorized -> Finance logs the loss.

### D. DELIVERY FAILURE (PROOF OF ATTEMPT)
- **Action:** Driver arrives at location. Customer is absent.
- **Process:** Driver takes a photo of the building and tags geolocation.
- **Notification:** Triggered to customer: "We missed you! [Photo Link] Click here to reschedule."
