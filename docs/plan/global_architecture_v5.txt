4. OPERATIONAL SAFETY PROTOCOLS (v5.0)
================================================================================

### A. DUAL-SCAN CUSTODY TRANSFER (The Handover Patch)
- **Protocol:** To move an item from `DRIVER` to `FACTORY`, both must scan the Bag ID.
- **Validation:** Verification of 10-meter geofence proximity (via Wi-Fi/GPS) before the transfer is committed to the DB.
- **Override:** Senior Admins can trigger a "Privilege Override" (logged as an incident).

### B. OFFLINE-FIRST ARCHITECTURE (Resilience)
- **Strategy:** Workbox.js captures all scans/data in `IndexedDB`.
- **Security:** Data is encrypted via **SubtleCrypto** using a session-derived key before storage.
- **Sync:** Background workers flush data to a **Redis Write-Buffer** once online, which is then batched into PostgreSQL.

### C. UNIFIED NOTIFICATION CENTER (The "Unified Inbox")
- **Frontend:** A centralized Bell Icon in the global header.
- **Abstraction:** Routes alerts to WhatsApp, SMS, Email, or In-App based on user `NotificationPreferences`.

### D. DEEP RTL ARCHITECTURE
- **Technical:** `tailwind-rtl` + Direction Context. All UI elements (shadows, padding, icons) physically flip for the Arabic locale.
