  - homepage uses client-side `/api/auth/me` redirect

  - 65 route files under `src/app/api`
  - only 6 route files reference `getCurrentUser`
  - high-value routes such as dashboard/assets/tickets/inventory/staff appear unauthenticated at handler level
- Main app architecture is a client-side SPA shell using Zustand `currentView` switching inside `src/app/page.tsx`
- Repository still contains substantial cross-domain carryover:
  - non-IT component folders (laundry/customer/transport/employee/etc.)
  - matching non-IT API areas (`business`, `complaints`, `customers`, `feedback`, `orders`, `chat`)
- Portal drift:
  - `customer-portal` uses Next 13 + React 18 and runs on port 3005
  - `internal-portal` uses Next 16 + React 18 + Prisma 4, and its root page currently returns `null`
- Overall assessment: strong core CRUD/data model foundation, but weak operational hardening and repo hygiene
